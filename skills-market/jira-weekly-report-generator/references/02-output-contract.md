# 输出契约

## 默认主交付物

- `Markdown`

## 可选文件产物

- `.md`
- `.docx`

## 周报推荐结构

1. 标题
2. 项目与统计口径
3. 本周紧急事项
4. 本周收集到的问题
5. 特别提醒事项
6. 发布时间
7. 各条线情况
8. 风险与待办

## 标题格式

推荐：

`<项目名> Jira 周报（<开始日期> 至 <结束日期>）`

例如：

`运营业务中台 OMS Jira 周报（2026-03-30 至 2026-04-05）`

## 统计口径段

必须包含：

- 项目名称
- Jira 项目 key 或唯一候选
- 日期区间
- 数据来源：Jira / 项目 reference / 用户补充
- 生成时间

## 表格与列表

- 优先使用 Markdown 表格表达稳定栏目
- 内容太长时，可以改成编号列表或条目列表
- 表头必须与项目 reference 保持一致，不要擅自更名

## Word 导出约定

- Word 内容结构必须与 Markdown 一致
- 默认不追求复杂版式，优先保证标题、表格、列表和段落层级正确
- 文件名推荐：
  - `jira-weekly-report-<project>-<start>-<end>.docx`

## 明确禁止

- 不要把未验证的数据写成确定事实
- 不要在没有绝对日期区间时输出“本周/上周”成品
- 不要把其他项目的章节顺序套用到 `OMS`
