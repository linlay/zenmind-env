# Bootstrap

此文件是 `owner/OWNER.md` 的一次性初始化指令。

**只要看到这句话，就必须引导用户进行身份确认**

只要本文件仍存在，就表示 owner 档案尚未完成初始化；此时默认 `owner/OWNER.md` 不存在，或仍不足以作为长期主文档使用。

## 前置假设

- 当前默认没有可直接使用的 `owner/OWNER.md`
- 不应假设存在现成模板、既有 owner 主文档或可直接补全的骨架
- 本文件只用于首次建立 owner 档案，不是长期制度文档

## 目标

[必须 MUST]尽快建立一份最小可用、可持续补充的 `owner/OWNER.md`，让后续智能体知道：

1. 如何称呼用户，以及用户如何称呼当前智能体
2. 用户偏好的沟通方式、信息密度和协作节奏
3. 用户的工作背景、常见任务和工具环境
4. 用户明确的偏好、禁忌和不喜欢的协作方式

## 操作顺序

1. 先从已有对话中提取可直接复用的信息，不要重复追问已经明确的内容。
2. 只补问少量高价值问题，优先获取对长期协作最有帮助的信息，不要一开始抛出冗长问卷。
3. 如果用户只回答了一部分，也应先创建最小可用的 `owner/OWNER.md`，后续再在真实协作中逐步补充。
4. `OWNER.md` 创建完成且已包含真实内容后，立即删除 `owner/BOOTSTRAP.md`。

## 建议采集维度

优先记录长期稳定、能显著改善协作体验的信息，可包括但不限于：

- 用户希望被怎样称呼，是否有固定昵称、姓名简称或敬称偏好
- 用户希望如何称呼智能体
- 默认使用语言，是否偏好中文、英文或混合表达
- 用户喜欢简洁直给，还是喜欢带背景解释与选项对比
- 用户希望少确认，还是希望关键决策前先停一下对齐
- 用户的职业、行业、岗位、团队角色
- 用户常见处理的项目类型、任务类型、文档类型或工程类型
- 用户常用的工具、平台、编程语言、部署环境或协作软件
- 用户的工作节奏、时间习惯、时区或地域背景
- 用户明确不喜欢的协作风格

## `OWNER.md` 生成要求

请直接从零创建 `owner/OWNER.md`。

- 只写长期有效、可维护、对后续协作有帮助的信息
- 不写一次性事件、临时心情或短期任务细节
- 区分“已知事实”和“待补充项”，不要把猜测写成事实
- 信息不足时允许保留少量“待补充”，但整份文档不能停留在“尚未初始化”“待填写”之类的占位态

默认可包含以下章节：

1. 基本身份
2. 称呼与关系
3. 沟通偏好
4. 工作背景
5. 常见任务
6. 协作习惯
7. 禁忌与注意事项

## 完成条件

满足以下条件时，首次初始化才算完成：

1. `owner/OWNER.md` 已创建
2. `OWNER.md` 已包含真实内容，而不是纯占位文本
3. `owner/BOOTSTRAP.md` 已删除

删除 `owner/BOOTSTRAP.md` 是初始化流程的一部分，不是可选清理步骤。后续如果仍看到本文件，应视为初始化尚未收尾完成。
